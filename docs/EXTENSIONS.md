# Phonograph Extensions

Source-of-truth spec for the extension framework. Implementation-focused; read
alongside `architecture.md` (domain model) and `CLAUDE.md` (invariants).

## Overview

Extensions are **independent external HTTP services** registered **per-user** via a
**manifest URL**. Phonograph is only a *client*: it fetches the manifest, validates
it, has the user approve the requested permissions, then calls the extension's
endpoint to **enrich track metadata** (the `Refresh Metadata` action).

Two hard rules, enforced structurally:

- **Extensions never run code inside Phonograph.** Every interaction is an outbound
  HTTP call; responses are parsed into strict Pydantic models and never executed.
- **Extensions never touch audio.** Only an allowlist of metadata fields is ever
  sent, and responses are parsed with `extra="forbid"` so nothing else (no bytes,
  no file paths) can cross the boundary.

## Permission model

| Permission | Grants |
|---|---|
| `read:tracks` | Send track metadata (artist, title, album, year, isrc) to the extension. |
| `read:libraries` | Send library names/structure to the extension. |
| `write:streaming_links` | Add/update `StreamingLink` rows on the user's tracks. |
| `write:track_metadata` | Fill missing `genre`/`year` on the user's tracks (non-destructive). |

Granted set **must equal** the manifest's `required_permissions` at install.
Enforced twice: when building the request (omit data for ungranted `read:*`) and
when applying the response (drop fields for ungranted `write:*`). No audio field is
ever sent or accepted.

## API contract (Phonograph â†’ extension)

All calls carry signature headers so the extension can verify the caller:

```
X-Phonograph-Extension: <extension id>
X-Phonograph-Timestamp: <unix seconds>
X-Phonograph-Signature: hex( HMAC-SHA256(shared_secret, "{timestamp}.{raw_body}") )
```

`shared_secret` is generated by Phonograph at install and shared via the optional
install handshake. It is never exposed through the Phonograph API.

**`POST {endpoint_url}/refresh`** â€” request:
```json
{
  "api_version": "1.0",
  "extension_id": 42,
  "granted_permissions": ["read:tracks", "write:streaming_links"],
  "tracks": [
    {"ref": 1001, "artist": "Radiohead", "title": "Idioteque",
     "album": "Kid A", "year": 2000, "isrc": null}
  ]
}
```
response (strict; unknown fields rejected; each item parsed independently):
```json
{
  "results": [
    {"ref": 1001,
     "streaming_links": [{"service": "spotify", "url": "https://open.spotify.com/track/..."}],
     "genre": "Electronic", "year": 2000}
  ]
}
```
`ref` echoes the track id sent. `service` must be one of Phonograph's known
streaming services or the link is skipped.

**Optional handshakes** â€” `POST {endpoint_url}/install` and `POST {endpoint_url}/uninstall`:
```json
{"extension_id": 42, "shared_secret": "â€¦", "granted_permissions": ["â€¦"]}
```
Return `200`. If unimplemented, the lifecycle action still succeeds (best-effort;
recorded as an event).

Response rules for every call: `Content-Type: application/json`, body â‰¤

## Extension lifecycle

```
install â†’ enable â‡„ disable â†’ update â†’ refresh â†’ remove
                     â”‚
                     â””â”€ update with new perms â†’ needs_reapproval â†’ reapprove
```

- **install** â€” fetch+validate manifest, approve permissions, generate secret,
  optional `/install` handshake; status `enabled`.
- **enable / disable** â€” toggle; disabled extensions cannot refresh.
- **update** â€” re-fetch manifest, update in place. New permissions set
  `needs_reapproval` and block refresh until re-approved.
- **refresh** â€” only when `enabled` and not `needs_reapproval`; sends permitted
  track data, applies permitted enrichment, sets `last_refresh_at`. On failure:
  status `error` + `last_error`.
- **remove** â€” optional `/uninstall` handshake, then delete. Metadata already
  written to tracks is **kept**.

## Required Phonograph endpoints (`/api/extensions`, approved users, owner-scoped)

| Method & path | Purpose |
|---|---|
| `POST /preview` | Validate a manifest URL, return preview (no persistence). |
| `POST ` (root) | Install `{manifest_url, approved_permissions}`. |
| `GET ` (root) | List the caller's extensions. |
| `GET /{id}` | Detail + recent events. |
| `POST /{id}/enable` Â· `POST /{id}/disable` | Toggle status. |
| `POST /{id}/refresh` | Run refresh, return summary. |
| `POST /{id}/update` | Re-fetch manifest. |
| `POST /{id}/reapprove` | Approve newly requested permissions. |
| `DELETE /{id}` | Remove. |

Cross-owner access returns `404`. Errors: invalid manifest `422`, bad state
(disabled / needs re-approval / cooldown) `409`, extension unreachable `502`.

## Security summary

HTTPS-only, SSRF-guarded egress (block private/loopback/link-local/metadata IPs,
re-checked on every call), JSON-only with size/timeout caps, strict parsing (no
code execution), HMAC-signed requests, secrets never returned, per-extension
refresh cooldown. Config lives in `backend/.env` (`EXTENSION_*`, see `.env.example`).
